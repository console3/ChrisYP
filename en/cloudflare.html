üîç Verification Type Description

Type 1: Cookies Mode

Characteristics: Normal mode returns cookies like cf_clearance, __cf_bm, etc.

  • When no redirect page appears but cf_clearance exists, you can set alpha=true for invisible verification
  • Proxy is required

Type 2: Turnstile CAPTCHA Mode

Characteristics: CAPTCHA style, returns cf_turnstile-response

  • Credits consumption: 300 points (150 points with proxy)
  • Usually embedded in login forms with verification parameters like captcha_api_key
  • Parameter values start with 0.
  • Proxy is optional

üîç Action/Cdata Parameter Finding (Advanced Feature)

When verification type is turnstile and render=explicit parameter is found in the target website's *turnstile/v0/api.js link:

Step 1: Set explicit=true

Step 2: Find action and cdata parameters

  • Open F12 and search for window.turnstile.render
  • Set breakpoint there, clear cookie cache and refresh page
  • Check the action value and fill it in
