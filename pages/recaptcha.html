📋 常见问题解答

如何区分 ReCaptcha v2 和 v3？

与其他平台不同，我们只区分普通版和企业版：

v3 特征：

  • 到 reload 接口就结束
  • size 参数通常为 invisible
  • 需要传入 action 参数
  • 查找方法：打开 F12 搜索 grecaptcha.execute，找到函数入参对象的 action 值

v2 特征：

  • 通过 reload 接口后还需请求 userverify 接口
  • size 通常为 normal
  • 不需要 action 参数

如何区分普通版和企业版？

  • 普通版：anchor 接口路由为 /recaptcha/api2/anchor
  • 企业版：anchor 接口路由为 /recaptcha/enterprise/anchor

企业版带 s 值（如 steam）的token为什么无法使用？

这与代理质量相关，建议：

  1. 先尝试使用本机IP
  2. 更换高质量代理
  3. 如仍无法使用，请联系客服

请求头参数

参数名 说明 必填
User-Token 用户密钥，从主页获取 ✅
Content-Type application/json ✅
Developer-Id 开发者ID，开发者用户使用，用户主页邀请链接的字符串(如 xxx/register?c=abcdef, 则 abcdef 为开发者ID) ❌

POST请求参数（JSON格式）

参数名 类型 必填 说明
sitekey String 必填 谷歌验证码对接key
• 从anchor/reload接口的k值获取
• 示例：6Le-wvkSAAAAAPBMRTvw0Q4Muexq9bi0DJwx_mJ-
• 获取方法：F12搜索"anchor"接口，查看k参数
referer String 必填 🚨 触发页面地址
• 请复制浏览器完整地址，不要修改
• 示例：https://www.example.com/login?id=123
• 注意：必须包含完整的URL参数和锚点
size String 必填 验证类型
• invisible：隐形验证（v3常用）
• normal：普通验证（v2常用）
• compact：紧凑型验证
• 必须与anchor接口的size值对应
title String 必填 触发页面的标题
• 获取方法：F12控制台输入 document.title
• 示例：Login - Example Site
• 注意：必须是实际页面标题，不能随意填写
action String v3必填 v3版本需要的action值
• 获取方法：搜索 grecaptcha.execute
• 示例：login、submit、homepage
• 企业版搜索：grecaptcha.enterprise.execute
• v2版本不需要此参数
proxy String 可选 代理地址
• HTTP代理：ip:port 或 user:pass@ip:port
• SOCKS5代理：socks5://ip:port
• 示例：192.168.1.1:8080
• 使用代理可享受优惠价格（150-300点数）
ubd Boolean 可选 是否为特殊ubd类型路由
• 默认值：false
• 检查方法：查看验证路由是否包含"ubd"
• 如果anchor接口路由包含ubd，设置为true
s String Steam专用 Steam的s值
• 仅Steam网站需要
• 从Steam验证接口获取
• 一般情况下不需要填写
• 示例：steam_s_value_here
sa String 可选 个别网站anchor接口路由的sa值
• 企业版可能需要
• 从anchor接口URL参数获取
• 示例：sa_parameter_value
• 大多数网站不需要此参数
hl String 可选 语言代码
• 默认：zh-CN（中文）
• 其他语言：en、ja、ko等
• 从anchor接口的hl参数获取
• 如果是zh-CN可以不填

🔍 参数获取方法

开发者工具手动获取

步骤1：获取基础参数

  • 搜索 anchor 接口，获取 k、size、hl 参数
  • k 值即为 sitekey
  • 如果 hl 为 zh-CN 可不填

步骤2：获取页面信息

  • referer：直接复制浏览器地址栏完整URL
  • title：F12控制台输入 document.title 获取

步骤3：获取action参数（仅v3需要）

方法1：搜索代码

  • 通用版搜索 .execute(
  • 企业版搜索 .enterprise.execute
  • 查找代码中的 action 参数

方法2：断点调试

  • 通用版输入 debug(grecaptcha.execute)
  • 企业版输入 debug(grecaptcha.enterprise.execute)
  • 触发验证后在Scope中复制action值

步骤4：检查UBD类型

  • 检查验证路由是否为 ubd 类型
  • 如果是，设置 ubd 参数为 true
